**Organisation**: Tasks are grouped by user story. Setup + Foundational (Phase 1 + 2) build the shared Zod schemas, the pure helper (`append-edit.ts`), and the module scaffold. User stories then layer the handler logic incrementally — US1 ships the happy-path MVP for `target_mode: "specific"` (both locator shapes, fs read-modify-write, EXTERNAL_EDITOR_CONFLICT classification folded in, registration); US2 layers in the diagnostic-error coverage (`NOTE_NOT_FOUND`, `CONTENT_EMPTY`, bracket-rejection, locator-mutex, external-editor-conflict, distinguish-from-FS_WRITE_FAILED); US3 adds dedicated test coverage for the inline opt-in; US4 adds active-mode focused-file resolution.

**Cohort divergences from sibling BI-043 (patch_block)** worth noting upfront:
- Tool name `append_note`, not `append` — ADR-010 N/A because the wrapper is fs-direct (R1), not a wrap of the upstream `obsidian append` subcommand. Cohort precedent: `write_note` doesn't mirror upstream `create`.
- Four-file module shape (one pure helper `append-edit.ts`), not five — end-of-file append needs no in-document scan, so no block-scan/heading-walk equivalent. Cohort precedent: `delete/`, `rename/`, `move/` ship a three-file shape with no helper.
- One new `details.code` (`CONTENT_EMPTY` under `VALIDATION_ERROR`, single state, no `details.reason`), not three. Bracket rejection (FR-001a) surfaces via the cohort's standard Zod-issue-path channel — no new sub-code introduced.
- US ordering reflects the spec's four-story shape: US1 (P1 named-locator + default separator + response shape), US2 (P2 typed failures), US3 (P2 inline opt-in), US4 (P3 active mode). US2 and US3 share the P2 tier — independent layers on top of US1.
- Active mode requires NO explicit opt-in flag (FR-004a, deliberate cohort exception to `write_note`'s mandatory `overwrite: true`, settled at Clarifications Q3).
- `EXTERNAL_EDITOR_CONFLICT` classification (FR-022) folded into US1's atomic-write task (T011) rather than split into a separate phase — cohort precedent from BI-043 T014.
- No race-detection task — neither the spec nor research.md publishes an `APPEND_RACE` discriminator; FR-021's single-invocation atomicity is the only race-adjacent contract, inherited from the substrate.
